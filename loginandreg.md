
### 1. Password Security (BCrypt)
- Passwords haru chai hashed using BCrypt with a work factor of 10
- Implementation in `EncryptionServiceImpl`:
  ```java
  private final BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();
  
  public String hashPassword(String password) {
      return passwordEncoder.encode(password);
  }
  
  public boolean verifyPassword(String rawPassword, String hashedPassword) {
      return passwordEncoder.matches(rawPassword, hashedPassword);
  }
  ```
- Faida haru:
  - Salt chai automatically generated and stored with the hash
  - Work factor makes brute-force attacks computationally expensive
  - aaba hashing algorithm ta vaihalyo

### 2. Data Encryption (AES-256)
- Sensitive data is encrypted using AES-256
- Implementation in `EncryptionServiceImpl`:
  ```java
  private final byte[] encryptionKey;
  private static final String ALGORITHM = "AES";
  
  public String encrypt(String data) {
      SecretKeySpec secretKey = new SecretKeySpec(encryptionKey, ALGORITHM);
      Cipher cipher = Cipher.getInstance(ALGORITHM);
      cipher.init(Cipher.ENCRYPT_MODE, secretKey);
      byte[] encryptedBytes = cipher.doFinal(data.getBytes());
      return Base64.getEncoder().encodeToString(encryptedBytes);
  }
  ```
- Configuration in `application.properties`:
  ```properties
  app.encryption.key=MTIzNDU2Nzg5MDEyMzQ1Njc4OTAxMjM0NTY3ODkwMTI=
  app.encryption.algorithm=AES
  ```
- FAIDA:
  - 256-bit key strength for strong encryption
  - Base64 encoding for safe storage
  - Symmetric encryption for efficient performance

## Database Structure
- Uses H2 database in file mode (`./data/securetransfer.mv.db`)
- Main table: `users`
  ```sql
  CREATE TABLE users (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY,
      username VARCHAR(255) NOT NULL UNIQUE,
      password_hash VARCHAR(255) NOT NULL,
      device_id VARCHAR(255),
      last_login TIMESTAMP,
      user_token VARCHAR(255) NOT NULL UNIQUE,
      created_at TIMESTAMP NOT NULL,
      active BOOLEAN NOT NULL,
      PRIMARY KEY (id)
  );
  ```


### 2. Login Process
1. User enters credentials
2. Security checks:
   - Username existence verification
   - Password verification using BCrypt
   - Device ID validation
3. On successful authentication:
   - Updates last login timestamp
   - Generates new session token
   - Sets active status
   - Loads main application screen
